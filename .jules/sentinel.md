## 2025-02-28 - [Information Leakage via Raw Error Responses]
**Vulnerability:** The application was exposing `String(error)` in the JSON response of the `/auth/register` endpoint (and potentially wrapping errors via `new Error(...)` in repository files which obfuscated original stack traces but still allowed for raw error messages to be bubbled up).
**Learning:** This is a codebase-specific security gap where error handling across the repository layer and controller layer either obfuscated the original stack trace (bad for internal logging) or inadvertently passed the raw stringified error to the end user (bad for security, leading to information leakage).
**Prevention:** In repository-layer files, always prefer `throw error;` directly rather than stringifying/wrapping it in `throw new Error(String(error))`. At the controller layer, log the error internally (`console.error(error)`) but ensure that client-facing responses only contain generic error messages like `{ message: "Registration failed" }` without raw error details.
